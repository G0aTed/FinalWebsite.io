Hypertext Transfer Protocol Secure

What is HTTPS?

Hypertext Transfer Protocol Secure is a Hypertext Transfer Protocol extension. It is widely used on the Internet for secure communication over a computer network. HTTPS encrypts the communication protocol using Transport Layer Security.

The primary motivations for HTTPS are website authentication and the protection of the privacy and integrity of data exchanged while in transit. It protects against man-in-the-middle attacks, and bidirectional encryption protects communications between a client and server from eavesdropping and tampering.

HTTPS's security is based on the underlying TLS, which typically uses long-term public and private keys to generate a short-term session key, which is then used to encrypt data flow between the client and the server.

Plaintext, Ciphertext and Cryptography

Ciphertext is encrypted text that has been converted from plaintext by an encryption algorithm. Ciphertext cannot be read until it has been decrypted. The decryption cipher is an algorithm that converts ciphertext to plaintext.

In data transimssion, To encrypt communications, HTTPS employs an encryption protocol, known as Transport Layer Security, but it was previously known as Secure Sockets Layer. This protocol protects communications by utilizing an asymmetric public key infrastructure.

HTTPS first encrypts the data it is being transferred, then the ciphertext is sent to your device. Once recieved, your device, using a key will then decrypt the ciphertext, turning it back into plaintext for you to view.

Asymmetric Encryption

Asymmetric encryption encrypts and decrypts data using two distinct but mathematically linked cryptographic keys. These keys are referred to as a "Public Key" and a "Private Key." They are known as a 'Public and Private Key Pair.'

A cryptographic algorithm is at the heart of Asymmetric Encryption. To generate a key pair, this algorithm employs a key generation protocol. Both keys are mathematically linked to one another. The relationship between the keys varies depending on the algorithm.

When one visits an HTTPS website/page,  the browser creates an Asymmetrically encrypted connection with that website. The public key of the SSL/TLS certificate installed on the website is automatically deduced by the aforementioned browser.

Symmetric Encryption

Symmetric encryption is a type of encryption that uses the same key to encrypt and decrypt electronic data. The key must be exchanged between the entities communicating using symmetric encryption so that it can be used in the decryption process.

While symmetric encryption is an older method of encryption, it is faster and more efficient than asymmetric encryption, which strains networks due to data size performance issues and heavy CPU use. Because symmetric encryption performs better and faster than asymmetric encryption, it is commonly used for bulk encryption / encrypting large amounts of data.

Secure Sockets Layer (SSL)

SSL, Secure Sockets Layer, is a protocol for establishing authenticated and encrypted links between networked computers. The protocol is widely used in applications such as email, instant messaging, and voice over IP, but its most visible use is in securing HTTPS.

SSL certificates help to keep online interactions private and reassure users that the website is genuine and safe to share personal information with. More importantly, an SSL certificate is required for an HTTPS web address, which means that HTTPS websites use SSL to encrypt their traffic.
